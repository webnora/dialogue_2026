## alina/cleaningTheGuard (1).ipynb

**Назначение:** Очистка и предварительная обработка текстовых данных из статей "The Guardian".

**Библиотеки:** `pandas`, `ast`, `BeautifulSoup`, `re`, `tqdm`.

**Входные данные:** `combined_guardian.csv`.

**Выходные данные:** `cleaned_combined_guardian.csv`.

**Процесс:**
1. Загружает CSV-файл с названием `combined_guardian.csv`.
2. Определяет функцию `clean_fields`, которая:
    * Извлекает текст из HTML-тела с помощью `BeautifulSoup`.
    * Удаляет URL-адреса и специальные символы с помощью регулярных выражений.
    * Нормализует пробелы и преобразует текст в нижний регистр.
3. Применяет эту функцию очистки к столбцу "fields" DataFrame.
4. Сохраняет очищенные данные в `cleaned_combined_guardian.csv`.

## alina/Obuchenie (2).ipynb

**Назначение:** Обучение модели классификации текста (вероятно, для классификации по жанрам или категориям) с использованием очищенных данных из предыдущего ноутбука. "Обучение" - русское слово, означающее "Training".

**Библиотеки:** `pandas`, `torch`, `sklearn`, `transformers` (`BertTokenizer`, `BertForSequenceClassification`), `datasets`, `tqdm`, `numpy`, `joblib`.

**Входные данные:** `cleaned_combined_guardian.csv`.

**Выходные данные:** Обученная модель BERT, сохраненная в `./bert_category_classifier`, токенизатор и `label_encoder.pkl`.

**Процесс:**
1. Загружает `cleaned_combined_guardian.csv`.
2. Выполняет дальнейшую очистку данных, удаляя строки с пропущенными значениями.
3. Кодирует текстовые категории в числовые метки.
4. Разделяет данные на обучающий, проверочный и тестовый наборы.
5. Использует `BertTokenizer` для токенизации текстовых данных.
6. Создает модель `BertForSequenceClassification`.
7. Настраивает оптимизатор (`AdamW`) и планировщик скорости обучения.
8. Обучает модель в течение 3 эпох, проверяя после каждой эпохи и сохраняя модель с наилучшими показателями.
9. Тестирует окончательную модель на тестовом наборе.
10. Сохраняет обученную модель, токенизатор и кодировщик меток.

## alina/WORKing (3)-Copy1.ipynb

**Назначение:** Этот ноутбук предназначен для извлечения статей из The Guardian API.

**Библиотеки:** `requests`, `pandas`, `time`.

**Выходные данные:** `guardian_non_news.csv`.

**Процесс:**
1. Настраивает ключ API и базовый URL для контентного API The Guardian.
2. Определяет конфигурации для извлечения различных типов статей: "Analytical", "Feature", "Editorial" и "Review".
3. Проходит по конфигурациям и страницам API для загрузки статей.
4. Добавляет "категорию" к каждой статье на основе конфигурации.
5. Сохраняет собранные статьи в CSV-файл с названием `guardian_non_news.csv`.
6. Также содержит код для загрузки и отображения данных из csv.
